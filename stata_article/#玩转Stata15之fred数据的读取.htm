在之前有读者问到关于fred数据的读取，在Stata14中我们可以用到命令 freduse 进行读取。该命令属于外部命令，需要键入如下命令行进行下载：

ssc install freduse

我们用到help文件中的例子，读取数据集GS1M、 CPN3M和 GS5。键入如下命令：

freduse GS1M CPN3M GS5

得到结果如下所示：

可以看到结果窗口中显示3个数据集中的数据都被读入，并且数据窗口也显示到已经直接将3个数据集的数据合并在一起了，接下来就可以根据自己的实际需求，对数据进行整理分析了。

尽管freduse可以读取fred数据，但前提是你必须知道具体的数据集名称。就在前几天发布的Stata15中，对于fred数据的读取，更是技高一筹。在Stata15的菜单式操作中，我们可以直接通过 File—Import—Federal Reserve Economic Data(FRED) ，进入fred数据的搜索页面，根据不同的需求输入需要搜索数据集的关键词进行搜索。但是读入fred数据需要一个 FRED key ，大家点击窗口下方的链接进行申请，只需要使用邮箱注册账号，填写一份简单的申请，当时就可以获得key。（不像某地图API需要等好几天进行验证……）

输入FRED key并点击OK，我们就进入到了 import fred 的 窗口。在这里，我们总结Stata官网(http://www.stata.com)上的例子进行说明。假如现在我们需要导入Venezuela、 Colombia和Peru的GDP数据，在fred数据搜索页面输入Venezuela Gross Domestic Product后点击search，得到如下页面：

可以看到弹出页面中有很多可供选择的数据，我们可以根据需求，将所需数据添加到页面的右侧，此例中我们添加数据RGDPNAVEA666NRUG。对于Colombia和Peru数据的获取，采取同样的方法，得到如下所示的页面：

然后，点击import后，弹出如下所示的页面：

在此我们可以选择数据的范围和导入的方式，最后点击submit，Stata将自动下载该数据并且保存在内存中，同时将所有的数据集合并在一起。

完成数据的导入，你就可以对数据进一步的分析整理了。以上菜单式操作完全可用命令 import fred 实现，我们读入数据后，Stata也自动帮我们生成了import fred的命令。这里需要 注意 的是，我们使用菜单式操作时已经输入了FRED key，并且永久保存，所以我们在使用import fred的时候不需要再设置FRED key了。但是如果我们没有通过窗口进行设置，在使用这个命令前需要用 set fredkey 命令来设置一下，加上permanently选项也可以帮我们永久保存FRED key。
